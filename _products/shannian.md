## 这个产品解决了什么？

你有没有过这样的经历：在走路、洗澡、开车的时候，突然想到一个绝妙的点子，但等到能拿出手机记录时，已经忘得差不多了？

闪念就是为了解决这个问题而诞生的。

通过 Apple Watch 或 iPhone 的语音输入，你可以在几秒钟内把想法记录下来。AI 会帮你把语音转成文字，整理成结构化的笔记。

- SwiftUI + SwiftData
- CloudKit 同步
- Speech Framework
- Whisper API（语音转文字）

## 遇到的问题 & 选择

### 为什么选择 SwiftData 而不是 Core Data？

一开始我用的是 Core Data，但维护成本太高了。迁移到 SwiftData 后，代码量减少了 40%，而且和 SwiftUI 的集成更自然。

当然，SwiftData 也有坑——比如复杂查询的性能问题。但对于闪念这种数据量不大的应用，利大于弊。

### 语音识别该用本地还是云端？

我尝试过 Apple 的 Speech Framework，识别率还可以，但对方言和口音支持不好。

最终选择了 Whisper API——识别率更高，支持的语言更多。代价是需要网络，而且有 API 成本。

这个决定现在看来是对的。用户反馈最多的就是"识别很准"。

### 做错了什么？

最大的错误是太早做了太多功能。

最初版本有标签、文件夹、搜索、导出、分享...结果用户根本用不到那么多功能，维护成本却很高。

后来我砍掉了一半的功能，专注于"快速记录"这个核心场景。反而收到了更好的反馈。
